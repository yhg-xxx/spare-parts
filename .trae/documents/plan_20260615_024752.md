## 计划：添加 Spring Boot Actuator 和 Prometheus 依赖及配置

根据你提供的图片配置，我需要对后端项目 `spare-parts-back` 做以下两处修改：

### 1. `spare-parts-back/pom.xml`
在 `<dependencies>` 节点中，在 `spring-boot-starter-web` 之后新增两个依赖：
- `spring-boot-starter-actuator`（Actuator 核心依赖）
- `micrometer-registry-prometheus`（暴露 Prometheus 指标格式的端点）

因为项目已继承 `spring-boot-starter-parent`，所以版本会由 Spring Boot 父 POM 自动管理，无需手动指定 `<version>`。

### 2. `spare-parts-back/src/main/resources/application.yml`
在文件末尾追加 `management` 配置：

```yaml
management:
  endpoints:
    web:
      exposure:
        include: health,info,prometheus
  endpoint:
    health:
      show-details: always
```

### 3. 关于 Spring Security 的注意事项
当前 `SecurityConfiguration.java` 只显式放行 `/login/**`、`/files/**` 等路径，**没有放行 `/actuator/**`**。如果直接启动，`/actuator/*` 会被 Spring Security 拦截而返回 401/403。

如果需要访问监控端点，建议在 `SecurityConfiguration.java:22-35` 的 `authorizeHttpRequests` 中再添加一行：
```java
.requestMatchers("/actuator/**").permitAll()
```
（如果你不希望做这一步，也可以告诉我，我就只按图片内容修改 pom 和 yml）

## 访问地址

启动后端后（默认端口 8080），可以通过以下地址查看监控数据：

| 地址 | 说明 |
| --- | --- |
| `http://localhost:8080/actuator` | Actuator 所有已暴露端点的总览（JSON 列表） |
| `http://localhost:8080/actuator/health` | 健康状态（UP/DOWN 及详情，已配置 `show-details: always`） |
| `http://localhost:8080/actuator/info` | 自定义应用信息（暂未配置 info.* 时返回空对象 `{}`） |
| `http://localhost:8080/actuator/prometheus` | **Prometheus 抓取目标**（返回 Prometheus 纯文本指标格式，供 Prometheus 服务器 `scrape_configs` 抓取） |

后续把 `http://localhost:8080/actuator/prometheus` 配置到 Prometheus 的 `prometheus.yml` 中即可接入 Grafana 进行可视化监控。
